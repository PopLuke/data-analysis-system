## 一、系统分析与设计

### 1.1 系统架构

本项目采用 **Flask 后端 + Jinja2 模板引擎 + ECharts 前端图表库** 的架构：

```
用户浏览器 → Flask 路由 → Pandas 数据处理 → JSON API → ECharts 图表渲染
```

- **后端：** Flask（Python Web框架）负责路由分发、Session管理、文件读写
- **数据处理：** Pandas 负责 CSV/Excel 文件加载、数据分组与聚合
- **前端图表：** ECharts 5（Apache 开源图表库）负责图表渲染与交互

### 1.2 数据处理流程

```
上传文件 → 数据加载 (utils/data_loader.py) → 数据清洗 (utils/cleaner.py)
→ 可视化 (本次负责) → 数据分析 (utils/analyzer.py) → 导出
```

## 三、负责的功能模块详细说明（可视化模块）

### 3.1 功能概述

可视化模块支持：
1. **4种图表类型**：柱状图、折线图、饼图、散点图（满足"3种以上"要求）
2. **用户自定义图表参数**：
   - 图表标题（自由输入）
   - 聚合方式（求和 / 均值 / 计数）
   - 颜色主题（默认 / 清新绿 / 暗黑系 / 复古风）
   - 显示/隐藏数据标签
   - 显示/隐藏图例

### 3.2 实现思路

**前后端分离的数据交互模式：**
- 前端通过 ECharts 渲染图表，用户配置参数后通过 Fetch API 发送 POST 请求到 `/api/chart-data`
- 后端接收参数（图表类型、X/Y轴字段、聚合方式），使用 Pandas 进行数据分组聚合，返回 JSON 格式的图表数据
- 前端根据返回数据构建 ECharts option 对象并渲染

**4种图表的处理差异：**
- **柱状图/折线图**：返回 `{x: [...], y: [...]}`，x为分组标签、y为聚合数值
- **饼图**：返回 `{data: [{name, value}, ...]}`，每个对象为一个扇区
- **散点图**：返回 `{data: [[x1,y1], [x2,y2], ...]}`，每个子数组为一个坐标点

### 3.3 后端 API 设计

**接口：** `POST /api/chart-data`

**请求参数（JSON）：**

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| chart_type | string | 是 | bar / line / pie / scatter |
| x_col | string | 是 | X轴/分组字段名 |
| y_col | string | 是 | Y轴/数值字段名 |
| agg_method | string | 否 | 聚合方式：sum(默认) / mean / count |

**返回格式（JSON）：**

```json
{
  "x": ["1月", "2月", ...],       // 柱状图/折线图：X轴标签
  "y": [4174, 4507, ...],         // 柱状图/折线图：Y轴数值
  "data": [{"name": "1月", "value": 4174}, ...],  // 饼图专用
  "data": [[x1, y1], ...],        // 散点图专用
  "chart_type": "bar",
  "data_count": 6
}
```

### 3.4 前端实现

**用户自定义参数控件清单：**

| 参数 | HTML控件 | 默认值 | 说明 |
|------|---------|--------|------|
| 图表标题 | `<input type="text">` | 空（自动生成） | 显示在图表顶部 |
| 图表类型 | `<select>` | 柱状图 | 4种可选 |
| X轴字段 | `<select>` | 第一个字段 | 从数据列动态生成 |
| Y轴字段 | `<select>` | 第一个数值字段 | 仅显示数值列 |
| 聚合方式 | `<select>` | 求和 | Sum/Mean/Count |
| 颜色主题 | `<select>` | 默认配色 | 4种预设配色方案 |
| 显示数据标签 | `<input type="checkbox">` | 关闭 | 在图表上显示数值 |
| 显示图例 | `<input type="checkbox">` | 开启 | 底部图例 |

**关键功能点：**
- 页面加载时自动生成默认图表（无需手动点击）
- 窗口缩放时图表自适应大小（`chart.resize()`）
- X轴标签过长时自动旋转避免重叠
- 折线图平滑曲线显示

### 3.5 主要代码片段

详见附录文件：`docs/code_snippets/visualization.py`

核心逻辑包括：
1. `_aggregate()` 函数：根据聚合方式对 DataFrame 执行 sum/mean/count 分组
2. `colorThemes` 对象：4种 ECharts 配色方案预设
3. 图表生成事件处理：读取参数 → 请求API → 构建option → 渲染

## 五、遇到的典型问题及解决方案

### 问题1：饼图和柱状图数据聚合方式单一

**现象：** 原有代码只有 sum（求和）聚合，对于需要看平均值或计数的场景无法满足。

**解决：** 在后端 API 新增 `agg_method` 参数，使用内部函数 `_aggregate()` 根据参数动态选择 `groupby().sum()` / `.mean()` / `.count()`，前端增加对应的下拉选择器。

### 问题2：窗口缩放时图表不变

**现象：** 浏览器窗口大小改变后，ECharts 图表不会自动适应新尺寸。

**解决：** 添加 `window.addEventListener('resize', () => chart.resize())` 监听窗口变化事件，触发 ECharts 实例的 resize 方法。

### 问题3：X轴标签过多时重叠

**现象：** 当分组数据量较大（>8个），X轴标签互相重叠无法辨认。

**解决：** 在 ECharts xAxis 配置中添加旋转逻辑：`axisLabel: { rotate: data.x.length > 8 ? 30 : 0 }`，超过8个标签时自动旋转30度。

## 六、个人收获与改进建议

### 6.1 个人收获

1. **前后端分离的数据可视化实践**：掌握了 Flask RESTful API + ECharts 的前后端交互模式，理解了如何将 Pandas 数据处理结果转换为图表库所需的数据格式
2. **ECharts 配置体系**：深入理解了 ECharts 的 option 配置结构（title、tooltip、legend、xAxis、yAxis、series），能够根据不同图表类型灵活配置
3. **用户体验设计**：通过增加自定义参数（标题、配色、标签开关），让图表从"固定展示"升级为"用户可配置"，提升了系统的灵活性
4. **Web开发规范**：学会了在团队项目中保持代码风格一致，使用 Jinja2 模板继承、Session管理、Flash消息等 Flask 生态

### 6.2 改进建议

1. **增加更多图表类型**：后续可加入雷达图、热力图、箱线图等，丰富可视化能力
2. **图表导出功能**：支持将 ECharts 图表导出为 PNG/SVG 图片，方便用户保存
3. **数据筛选过滤**：在可视化前增加数据过滤条件（如日期范围、数值阈值），提高灵活性
4. **多图表联动**：支持在一个页面同时展示多个图表，通过刷选联动分析
5. **图表配置持久化**：将用户的图表配置保存到 Session，刷新页面后保持之前的选择
